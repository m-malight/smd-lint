After Pasting on Hackmd:

  • Remove all spaces above and below the code block if there are any.
  • Ensure that each line starts from the leftmost screen column, including items in the list.

Code Blocks:

  • Each code block should be surrounded by triple backticks at the top and bottom.
  • The language should be specified at the beginning of each code block.
  • There shouldn't be anything else at the top of the code blocks.
  • No empty lines should be present at the top or bottom of code blocks.
  • Code blocks should start at the leftmost screen column.
  • Code blocks should be formatted using Prettier with an indent of 2.

Tech Words:

  • Technical words must always be in code style (codeWord).
  • Proper nouns and common English words must NOT be in code style.

Text Formatting:

  • Sentences should start with a capital letter.
  • Words in the middle of a sentence shouldn't start with a capital letter unless they are proper nouns.
  • Sentences must end with punctuation.
  • Paragraphs should be separated by an empty line.
  • Headings should use heading style (e.g., ## Heading) and not bold formatting.
  • Headings shouldn't be in all caps.
  • There should be only one level-1 (#) heading at the top of the document.
  • Level-3 (###) headings should have previous level-2 (##) headings.
  • Headings must be followed by some paragraph of text.

Lists:

  • Numbered lists should only be used for rankings or sequences.
  • Bullets should be used for other types of lists.
  • Bullets should not be overused.

Grammar and Typos:

  • Text should be checked for typos and grammatical issues using Grammarly.

Images:

  • Images should have introductions and explanations.
  • Images should be cropped to focus on relevant content.
  • Text in images should be easily readable.
  • Only relevant content should be included in images.
